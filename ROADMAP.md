# 仓库路线图与建议 · Roadmap & Suggestions

## 一、当前状态评估

### 优势
- 视觉完成度高：19+ 天体全程序化着色器，Bloom 后期，体积日冕
- 科学性达标：真实开普勒椭圆轨道 + 倾角 + 偏心率
- 工程整洁：模块化(HTML + 多 JS)，质量三档自动降级，移动端响应式
- 交互完整：聚焦/搜索/预设/成就/键盘/i18n
- 部署自动化：GitHub Actions → Pages

### 待补强(详见 REALISM.md)
- 全行星轴倾角、海卫一逆行渲染、严格开普勒第三定律

## 二、对仓库的具体建议

### A. 工程质量

1. **加 ESLint + Prettier 配置**
   - 当前纯手写，缺少统一风格校验
   - 加 `.eslintrc` + `.prettierrc`，CI 跑 lint

2. **加自动化测试**
   - 当前无测试。可加 Playwright 截图回归测试(首屏是否报错、关键天体是否渲染)
   - 至少加一个冒烟测试：`npm run smoke` 用 puppeteer 打开页面检查 console 无 error

3. **CI 加语法检查**
   - 在 deploy workflow 前加一步 `node --check` 全文件，防止语法错误部署上线

4. **加 `package.json`** 即便无构建
   - 声明 dev 依赖(eslint/playwright)、scripts(`dev`/`smoke`/`lint`)、engines 字段
   - 让贡献者一键起步

### B. 性能与体验

5. **加性能监控浮层**
   - 当前只有 FPS。可加 draw calls、triangles、GPU 内存(若可获取)
   - 低 FPS 时自动建议切换性能档

6. **资源懒加载**
   - 远处天体(柯伊伯带外)可用 LOD 降低分段，相机靠近时再提升
   - 当前全量 64 分段，LOD 可省 30-50% 三角面

7. **WebGL 上下文丢失处理**
   - 加 `renderer.context lost` 监听，提示用户而非静默崩溃

### C. 内容深度

8. **时间旅行模式**
   - 输入任意日期，行星定位到该时刻真实位置(用星历表近似)
   - 科普价值极高(如"1986 年哈雷彗星回归时行星在哪")

9. **哈雷彗星等著名彗星**
   - 加彗星轨道(高偏心率椭圆)+ 彗发/彗尾粒子(近日点激活)

10. **日食/月食模拟**
    - 地月太阳三星一线时检测食相，加阴影锥可视化

11. **航天器轨迹回放**
    - 新视野号、旅行者号的真实轨迹动画(数据 NASA Horizons 可查)

### D. 可访问性与国际化

12. **i18n 扩展到更多语言**(日/西/法)
    - 当前中英双语架构已就绪，加语言包即可

13. **屏幕阅读器支持**
    - 给天体加 ARIA 标签，键盘聚焦时朗读名称与数据

14. **色觉障碍友好**
    - 天体颜色加图案/亮度差异备选方案(非仅靠色相)

### E. 社区与文档

15. **CONTRIBUTING.md**
    - 说明如何添加新天体(数据格式 + 着色器模板)
    - 列出"待认领"任务(如"为某矮行星加着色器")

16. **在线演示 GIF/视频放 README**
    - 录一段 10 秒演示，比静态截图更吸引星标

17. **LICENSE 已有 MIT，加 CODE_OF_CONDUCT**

18. **GitHub Discussions 开启**
    - 收集用户反馈的天体观感问题

## 三、优先级排序(若继续投入)

| 优先级 | 建议 | 工作量 | 价值 |
|--------|------|--------|------|
| P0 | CI 语法检查(#3) | 0.5h | 防事故 |
| P0 | 冒烟测试(#2) | 2h | 防回归 |
| P1 | 全行星轴倾角 | 1h | 科学性 |
| P1 | 时间旅行模式(#8) | 1d | 杀手级功能 |
| P1 | 哈雷彗星(#9) | 4h | 高辨识度 |
| P2 | LOD(#6) | 1d | 性能 |
| P2 | 日食模拟(#10) | 1d | 难度高但惊艳 |
| P3 | 多语言扩展(#12) | 持续 | 国际化 |

## 四、不建议做的(避免过度工程)

- ❌ 不要加后端/数据库(违背"单文件运行"精神)
- ❌ 不要引入构建工具链(Vite/Webpack)——当前 importmap + CDN 零构建是优势
- ❌ 不要加用户账号/云存档(成就用 localStorage 够了)
- ❌ 不要为"未来可能的需求"预留接口(YAGNI)
